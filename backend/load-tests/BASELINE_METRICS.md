# 性能基线指标记录（任务 12.6）

本文档用于记录后端关键接口的性能基线，便于**回归追踪**：每次重大改动（尤其是
SSE 架构、数据库查询、缓存层）后重新跑一遍压测，把结果填入下表，对比是否退化。

> 单位说明：延迟均为毫秒（ms），吞吐量为每秒请求数（req/s）。
> 每次记录请同时填写「环境」一行，保证可比性（机器、是否本地、LLM 模型等）。

## 3. SSE 并发 + 首字内容延迟（任务 12.1 / 12.5）

脚本：`sse-concurrency.mjs`（内置 http）。命令示例：

```bash
CONCURRENCY=100 node backend/load-tests/sse-concurrency.mjs
```

- **TTFB**：发起请求到收到首字节（通常是 `:ok` 心跳）。反映连接建立 + 服务端非阻塞响应能力。
- **TTFC**：发起请求到首个 `content_delta` 事件。反映「首字内容延迟」，含 LLM 首 token 时间（任务 12.5）。
- **总耗时**：发起请求到 `message_complete`。

## 4. 回归判定建议

- p95 较上一次基线劣化 **> 20%** 时，视为潜在性能退化，需排查。
- SSE 成功率较基线下降 **> 1 个百分点** 时，需排查并发/资源泄漏问题。
- 每个脚本结尾会输出一行 `METRICS_JSON {...}`，可复制保存或用脚本提取归档。
