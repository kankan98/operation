### 失败类别 2: AI 响应超时（2 个测试）

**测试**:
- should send message and receive streaming response
- creates a conversation, streams a response

**原因**: 等待 AI 响应超时（默认 30-60 秒）

**分析**:
- 这些测试依赖真实的 AI API 调用
- AI 响应时间不可控
- 不适合作为自动化 E2E 测试

**修复方案**: 
1. 使用 mock AI 响应
2. 或标记为 `@slow` 测试，仅手动运行
3. 或增加超时到 5 分钟

### 失败类别 3: 测试逻辑问题（3 个测试）

**期望**: 双击只发送一次  
**实际**: 没有消息发送  
**原因**: 测试逻辑有问题，双击太快导致第一次点击也被防护

#### 测试 2: should prevent rapid Enter keypresses  
**期望**: 快速按 Enter 只发送一次  
**实际**: 没有消息发送（0 条用户消息）  
**原因**: Enter 事件可能被其他逻辑拦截

#### 测试 3: should disable input during streaming
**期望**: 流式传输期间输入框禁用  
**实际**: 输入框仍然启用  
**原因**: 
- AI 响应太快，测试来不及捕获禁用状态
- 或者输入框禁用逻辑未正确实现
