## Why

The repository now has routes, OpenSpec specs, a roadmap, and deployment notes, but the long-running autonomous development objective is still spread across conversation history and several documents. Future AI agents need one governed goal contract that explains how to keep improving the product for Chinese badminton racket live-commerce operators without bypassing OpenSpec, UX quality, research, security, or verification.

## What Changes

- Add a durable AI continuous development goal capability that defines the product target, target users, iteration loop, research policy, UX usefulness gate, documentation synchronization, and completion evidence.
- Add a project goal document that consolidates development tasks, roadmap lines, current standards, user collaboration points, and next iteration candidates.
- Update the autonomous roadmap and agent instructions to point future agents to the goal contract before starting self-directed work.
- Clarify that agents may research, use skills, install justified tools, or propose dependencies when uncertainty blocks progress, but durable decisions must be recorded in OpenSpec or project documentation.
- Keep this as a planning/governance change only: no backend, database, AI provider, RAG runtime, authentication, or new npm dependency is implemented.

## Capabilities

### New Capabilities

- `ai-continuous-development-goal`: Defines the governed long-running goal, autonomous iteration responsibilities, user-value checks, research and dependency guardrails, and evidence required before claiming the goal is complete.

### Modified Capabilities

- `continuous-improvement-roadmap`: Adds the project goal document as the source for self-directed iteration selection and roadmap synchronization.
- `ai-development-governance`: Adds the requirement that future autonomous development waves start from the goal contract and update it when new durable work, risk, or user collaboration needs are discovered.

## Impact

- Affected documentation: `docs/roadmap/`, `AGENTS.md`, `.codex/rules/`, and accepted OpenSpec specs.
- Affected workflow: future agents will use the goal document plus OpenSpec specs to choose and verify autonomous development waves.
- No runtime code, UI, API, database, AI provider, deployment target, or package dependency changes.
