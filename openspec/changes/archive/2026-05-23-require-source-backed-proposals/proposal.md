## Why

Future proposals need a reliable research step before scope is locked, so the
project does not build from assumptions, fake market signals, or generic AI
ideas. This is especially important for a badminton live-commerce operations
tool, where useful features depend on operator workflow, source reliability,
platform rules, and AI safety constraints.

## What Changes

- Require every non-trivial OpenSpec proposal to include a pre-proposal research
  pass using professional, official, primary, or otherwise credible sources.
- Require every non-trivial requirements or proposal start to use relevant
  discovery, product, UX, OpenSpec, or implementation skills to check whether
  the idea is valuable, aligned with the original product goal, realistic to
  build, and likely to meet or exceed operator expectations.
- Require proposals or designs to record what sources were checked, why they are
  considered reliable, and how they changed user value, scope, risks, or
  verification.
- Require the agent to reason from the operator’s point of view: which user role
  benefits, what work becomes easier, and what result the user can achieve.
- Require future proposals to distinguish useful product polish from decorative
  work: the work may create a memorable highlight, but it must stay calm,
  operational, accessible, and consistent with the code and UX baseline.
- Require agents to adjust active specs, rules, contracts, roadmap notes, or
  tasks during development when implementation evidence reveals business drift,
  unreasonable assumptions, user-value gaps, or conflicting guidance.
- Clarify that researched facts are not automatically product truth or AI
  grounding; reusable business/domain knowledge still enters the reviewable
  knowledge lifecycle.
- Update project rules, goal, and accepted governance specs.

## Capabilities

### New Capabilities

- None.

### Modified Capabilities

- `ai-development-governance`: Requires source-backed proposal research,
  skill-backed value exploration, and user-value framing before non-trivial
  proposal scope is finalized.
- `ai-continuous-development-goal`: Updates autonomous continuation expectations
  so future waves research reliable sources, use relevant discovery skills, and
  connect findings to user outcomes before proposal work.

## Impact

- Affected documentation: `AGENTS.md`, `.codex/rules/`, `docs/roadmap/`, and
  OpenSpec accepted specs after archive.
- Affected workflow: future proposal creation must include reliable source
  research before scope is finalized.
- No runtime code, UI behavior, database, API route, dependency, Docker image,
  or public preview change.
