## Context

`SessionGroupList` 目前对未命名会话使用同一个占位文案“新对话”展示标题，并将该文案拼进操作菜单按钮的 `aria-label`。线上 Playwright 复现显示，主“新对话”按钮和两个未命名会话操作菜单按钮会同时匹配 `getByRole('button', { name: '新对话' })`，造成严格模式点击失败，也降低了辅助技术用户区分控件的清晰度。

## Goals / Non-Goals

**Goals:**

- 保留未命名会话的视觉标题“新对话”。
- 让所有会话操作菜单按钮的可访问名称采用独立命名空间，避免与主新建会话按钮冲突。
- 用回归测试固定未命名会话菜单的可访问名称。

**Non-Goals:**

- 不改变会话标题数据、后端接口或数据库。
- 不调整会话列表视觉样式、分组、搜索或菜单内容。
- 不重构聊天页整体交互。

## Decisions

- 使用专门的菜单标签生成逻辑，而不是复用视觉标题拼接。
  - 方案 B: `会话操作菜单：${session.title || '未命名对话'}`。
  - 理由: 可访问名称先说明控件类型，再说明目标会话；未命名 fallback 不再包含主按钮完整名称“新对话”。
  - 替代方案 A: 改视觉标题为“未命名对话”。该方案会改变用户已熟悉的列表展示，不符合最小修复。
  - 替代方案 C: 在测试中使用 `exact: true` 或 test id。该方案只规避自动化问题，没有修复真实可访问名称冲突。
- 测试直接断言主“新对话”按钮可被精确定位并触发 `onNewSession`，同时断言未命名菜单使用新的可访问名称。

## Risks / Trade-offs

- 辅助技术听到的菜单名称会与屏幕上标题略有不同 -> 通过“未命名对话”表达真实状态，且保留目标上下文。
- 现有测试中按旧菜单名称查询的断言需要更新 -> 只调整会话菜单相关测试，避免扩大影响面。
- 线上仍可能存在旧构建缓存 -> 部署后用 Playwright 读取按钮 `aria-label` 集合并复测严格模式点击。
