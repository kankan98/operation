# Chat 用户体验问题修复总结

## 修复日期
2026-06-20

## 问题描述

用户反馈了两个主要问题：

1. **右侧工具执行状态与对话不一致**
   - 问题：右侧任务面板显示的工具调用与当前对话消息不匹配
   - 影响：用户无法准确了解当前对话中执行了哪些工具

2. **点击菜单栏"智能助手"应默认显示新对话**
   - 问题：点击后可能显示空白页面，用户不清楚当前状态
   - 影响：用户体验困惑，不知道是系统问题还是正常状态

## 修复方案

### 1. 工具执行状态修复 ✅

**根本原因**：
- 消息格式已升级为 `parts` 数组（按时序存储文本块和工具块）
- 但工具提取逻辑仍从旧的 `toolCalls` 字段读取
- 导致右侧面板无法显示新格式消息中的工具调用

**解决方案**：
```typescript
// 修改前
const toolExecutions = messages
  .flatMap((msg) => msg.toolCalls || [])
  .filter((toolCall) => toolCall.id);

// 修改后
const toolExecutions = messages
  .flatMap((msg) => {
    // 优先使用 parts 数组（新格式）
    if (msg.parts && msg.parts.length > 0) {
      return msg.parts
        .filter((part) => part.type === 'tool')
        .map((part) => ({
          id: part.id,
          name: part.name || '',
          input: part.input || ,
          result: part.result,
          isError: part.isError,
          startTime: part.startTime,
          endTime: part.endTime,
          durationMs: part.durationMs,
        }));
    }
    // 向后兼容：使用 toolCalls 字段（旧格式）
    return msg.toolCalls || [];
  })
  .filter((toolCall) => toolCall.id);
```

**修改文件**：
- `frontend/src/pages/Chat.tsx` (第60-80行)

**测试验证**：
- ✅ 新格式消息正确显示工具调用
- ✅ 旧格式消息向后兼容
- ✅ 切换会话时工具面板正确更新

## 验证步骤

### 验证工具执行状态
1. 启动服务：
   ```bash
   cd frontend && npm run dev
   cd backend && npm run dev
   ```

2. 访问 http://localhost:3003/chat

3. 发送消息："帮我读取 README.md 文件"

4. 等待 AI 响应和工具调用

5. 点击右侧"工具执行"标签

6. 验证：
   - ✅ 显示 "Read" 工具卡
   - ✅ 工具状态正确（成功/失败/运行中）
   - ✅ 点击工具卡可以跳转到对话中的对应位置

### 验证新对话欢迎页
1. 点击主导航的"智能助手"菜单

2. 验证：
   - ✅ URL 是 `/chat`（无 sessionId）
   - ✅ 显示欢迎页（紫色图标、标题、说明文本）
   - ✅ 显示4个快速开始按钮
   - ✅ 点击按钮会填充到输入框
   - ✅ 输入框可用且有焦点

3. 发送一条消息

4. 验证：
   - ✅ URL 更新为 `/chat/:sessionId`
   - ✅ 欢迎页消失，显示对话消息
   - ✅ 右侧面板显示对应的工具调用（如果有）

## 技术亮点

1. **向后兼容**
   - 同时支持新旧两种消息格式
   - 优雅降级，不影响历史数据

2. **状态管理清晰**
   - URL 是会话的唯一 source of truth
   - Store 不再持久化 currentSessionId
   - 避免缓存导致的导航问题

3. **用户体验优先**
   - 欢迎页提供明确的状态指示
   - 快速开始按钮降低使用门槛
   - 视觉设计统一、美观

4. **可测试性**
   - 完整的 E2E 测试覆盖
   - 调试工具辅助问题排查
