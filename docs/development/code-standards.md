# 代码规范总览

> **TL;DR**: 使用 TypeScript strict 模式。后端：2 空格缩进、单引号、分号。前端：函数组件 + Hooks、Tailwind CSS。命名：camelCase 变量/函数、PascalCase 类/组件、UPPER_SNAKE_CASE 常量、kebab-case 文件名。

详细规范：

- [前端代码约定](./frontend-conventions.md)
- [TypeScript 规范](./typescript-guidelines.md)
- [命名约定](./naming-conventions.md)
- [文件组织](./file-organization.md)

## 核心原则

1. **一致性** > 个人偏好
2. **可读性** > 简洁性
3. **类型安全** > 灵活性
4. **测试覆盖** > 快速实现
