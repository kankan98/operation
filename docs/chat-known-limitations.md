# Chat 已知问题和限制

> **TL;DR**: Chat 的核心会话、任务面板、工具卡和响应式流程已覆盖测试。仍需真实用户运行期监控、生产环境 AI provider 稳定性观察，以及后续独立 change 处理全站容器查询迁移。

---

## 当前限制

- 任务面板的“笔记”功能仍是未来扩展，不在本次变更范围内。
- 工具执行历史只展示当前会话消息中的工具调用；暂未提供独立历史查询 API。
- 会话搜索为基础文本搜索，匹配标题、预览和标签；暂不支持时间、平台、产品等高级过滤。
- 任务详情滚动依赖 `metadata.messageId`。没有该字段时会回退到任务卡自身锚点。
- Chat 已使用容器查询，但其他页面仍主要使用视口断点；全站迁移已列入 backlog。
- `26.1` 的“两周稳定性监控”是时间门槛任务，需要真实运行期数据后再关闭。

---

## 运行期观察点

- SSE 连接错误率和重连频率。
- `/api/chat/stream` 与 `/api/chat/streams/:streamId` 的 4xx/5xx。
- `/api/tasks/:sessionId` 查询耗时。
- 任务状态是否出现长时间停留在 `in_progress`。
- Chat 在 900/1280/1440/1920 视口宽度下是否持续无裁切。
- 新会话懒创建是否仍避免空壳会话。

---

## 回退策略

如果 Chat 出现阻塞问题：

1. 保留数据库字段和 API，不做数据回滚。
2. 临时隐藏入口或回滚前端构建。
3. 使用 Playwright E2E 复现并补回归测试。

旧版 Chat 组件已清理，本次变更不再维护 `/chat-legacy`。
