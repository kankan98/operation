# 后端架构

> **TL;DR**: 后端采用三层架构：Routes (HTTP 层) → Services (业务逻辑) → Database (数据访问)。职责分离清晰，Routes 处理请求响应，Services 包含核心逻辑，Database 层封装数据操作。新功能按层级依次添加。

---

## 分层架构图

```
┌─────────────────────────────────────────────────────────────┐
│ Routes Layer (路由层)                                       │
│ 职责：HTTP 请求/响应处理、参数验证、调用 Service           │
│ 禁止：业务逻辑、直接数据库操作                              │
├─────────────────────────────────────────────────────────────┤
│  routes/                                                    │
│  ├─ products.ts      → GET/POST/PUT/DELETE /api/products   │
│  ├─ alerts.ts        → 警报规则和警报历史 API              │
│  ├─ chat.ts          → 聊天会话和流式响应 API              │
│  └─ analytics.ts     → 价格分析和统计 API                  │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用
┌─────────────────────────────────────────────────────────────┐
│ Services Layer (服务层)                                     │
│ 职责：业务逻辑、数据处理、编排多个 DB 操作                 │
│ 禁止：HTTP 细节 (req/res)、直接返回 HTTP 响应              │
├─────────────────────────────────────────────────────────────┤
│  services/                                                  │
│  ├─ productService.ts    → 产品CRUD、监控逻辑              │
│  ├─ alertService.ts      → 警报规则引擎、触发检查          │
│  ├─ chatService.ts       → AI 聊天、流式响应封装           │
│  ├─ scraperService.ts    → 价格爬取逻辑                    │
│  └─ analyticsService.ts  → 价格统计、趋势分析              │
└─────────────────────────────────────────────────────────────┘
                          ↓ 调用
┌─────────────────────────────────────────────────────────────┐
│ Database Layer (数据层)                                     │
│ 职责：Schema 定义、ORM 查询、数据持久化                    │
│ 禁止：业务逻辑                                              │
├─────────────────────────────────────────────────────────────┤
│  db/                                                        │
│  ├─ schema.ts      → Drizzle schema 定义                   │
│  ├─ index.ts       → 数据库连接和导出                      │
│  └─ migrations/    → 数据库迁移文件                        │
└─────────────────────────────────────────────────────────────┘
```

## 新功能添加指南

### 添加新的 API 端点

**步骤**:

1. **定义 Schema** (db/schema.ts)
2. **创建 Service** (services/)
3. **创建 Routes** (routes/)
4. **注册路由** (app.ts)
5. **编写测试**
