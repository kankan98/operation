# 测试最佳实践

> 测试策略和验收标准指南

---

## 1. 自动化测试优先级

| 优先级 | 测试类型 | 是否必须自动化 | ROI |
|--------|---------|---------------|-----|
| P0 | 核心功能单元测试 | ✅ 必须 | 高 |
| P1 | 回归测试 | ✅ 必须 | 高 |
| P2 | 核心用户流程 E2E | ✅ 建议 | 中 |
| P3 | UI 交互测试 | ⚠️ 可手动 | 低 |
| P4 | 边界场景测试 | ⚠️ 可手动 | 低 |

**决策原则**：自动化收益 > 自动化成本 → 自动化

---

## 2. 多级验收标准

### Level 1: 基础验收（必须达到）

- ✅ 核心功能逻辑正确（代码审查 + 手动验证）
- ✅ 无语法错误，类型检查通过
- ✅ ESLint 通过（0 errors）
- ✅ 关键路径单元测试通过
- ✅ 无阻塞性 bug

**适用**：时间紧急，快速交付 MVP

### Level 2: 标准验收（应当达到）

- ✅ Level 1 所有项
- ✅ 单元测试覆盖率 ≥ 70%
- ✅ 核心 E2E 测试通过
- ✅ 无严重性 bug
- ✅ 代码质量良好

**适用**：正常开发节奏

### Level 3: 优秀验收（理想目标）

- ✅ Level 2 所有项
- ✅ 单元测试覆盖率 ≥ 80%
- ✅ 所有自动化测试通过
- ✅ 性能指标达标
- ✅ 完整文档

**适用**：重要功能、充足时间

## 3. 测试失败诊断

### 三种失败类型

**类型 1：代码逻辑错误**
- 特征：断言失败、逻辑条件错误
- 处理：修复代码，重新测试

**类型 2：测试代码问题**
- 特征：断言条件不正确、Mock 配置错误
- 处理：修复测试脚本

**类型 3：测试环境限制**
- 特征：代码正确、手动验证通过、自动化测试失败
- 处理：标记为"手动验证"，记录到 TESTING_REPORT.md

### 诊断决策树

```
测试失败
    ↓
代码审查：逻辑是否正确？
    ├─ 否 → 类型 1：代码错误
    └─ 是 → 继续判断
        ↓
手动验证：功能是否正常？
    ├─ 否 → 类型 1：代码错误（隐藏 bug）
    └─ 是 → 继续判断
        ↓
测试审查：预期是否合理？
    ├─ 否 → 类型 2：测试问题
    └─ 是 → 类型 3：环境限制
```

## 4. 测试工具选择

| 测试类型 | 推荐工具 | 优先级 |
|---------|---------|--------|
| 单元测试 | Jest / Vitest | P0 必须 |
| 集成测试 | Jest + Testing Library | P1 建议 |
| E2E 测试 | Playwright | P1 建议 |
| E2E 测试（备选）| Puppeteer | P2 可选 |

**原则**：
1. 优先使用项目已有工具
2. E2E 测试优先 Playwright
3. 测试数量适度（E2E 成本高）

---

## 5. 后台任务管理

| 任务类型 | 是否后台 | 原因 |
|---------|---------|------|
| 开发服务器 | ✅ | 长期运行 |
| 构建任务 | ❌ | 需要结果 |
| 依赖安装（大型） | ✅ | 可能很慢 |
| 单元测试 | ❌ | 快速执行 |
| E2E 测试（长时间） | ✅ | 数分钟 |

**优先级**：
- 关键路径任务：前台运行
- 并行任务：后台运行
- 验证任务：前台运行

## 6. 测试报告结构

```markdown
# 测试报告

## 测试概览
- 变更名称、测试日期、测试环境

## 测试执行情况
### 单元测试
- 用例数、通过数、失败数、覆盖率

### 集成测试
- 场景数、通过数、失败数

### E2E 测试
- 场景数、通过数、失败数

## 失败分析
| 测试项 | 失败原因 | 代码状态 | 手动验证 | 结论 |
|--------|---------|---------|---------|------|
| XX | 环境限制 | ✅ 正确 | ✅ 通过 | 接受失败 |

## 测试结论
- [ ] 所有测试通过
- [ ] 覆盖率达标
- [ ] 无阻塞性问题
- [ ] 可以发布
```

---

## 核心原则

1. **不盲目追求 100% 自动化** - 考虑 ROI
2. **不盲目追求 100% 通过率** - 区分代码问题和环境问题
3. **选择合适的验收标准** - Level 1/2/3 灵活选择
4. **善用后台任务** - 提升效率
5. **详细记录结果** - 便于追溯
