# 阶段 1：需求分析（Explore）

## 执行步骤

1. 使用 `/openspec-explore <change-name>` 启动分析
2. 搜索相关代码（限定 `src/` 目录，避免 `node_modules/`）
3. 读取关键文件，记录所有相关文件路径
4. 生成需求文档（路径由 openspec-explore 自动处理）

## proposal.md 必须包含

- 业务目标（核心价值、成功指标）
- 技术范围（涉及的模块/组件、相关文件清单）
- 技术约束（兼容性、性能、技术栈）
- 风险识别（至少 1 个风险，包含影响、概率、缓解措施）

## 自我验证清单

- [ ] proposal.md 已创建且非空
- [ ] 包含"业务目标"章节
- [ ] 包含"技术范围"章节
- [ ] 列出至少 3 个相关文件
- [ ] 识别至少 1 个风险或约束
- [ ] 所有内容使用中文编写

## 门禁 1

向用户展示分析结果，等待确认后进入阶段 2。

展示格式：
- 变更名称
- 核心目标（1-2 句）
- 技术范围（涉及模块数、文件数）
- 主要风险及缓解措施
- 完整文档路径

用户响应："确认" → 进入阶段 2 | "需要修改" → 询问并更新 | "查看文档" → 展示完整内容
